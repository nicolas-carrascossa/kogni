# User Flows

## Fluxo 1: visitante para signup

### Objetivo

Converter um visitante interessado em um usuário autenticado.

### Passos

1. Usuário chega na landing page.
2. Entende a proposta: company brain para alimentar ferramentas de IA.
3. Vê exemplos de fontes como GitHub e Slack.
4. Clica em CTA principal.
5. Vai para tela de login/signup.
6. Cria conta ou entra.
7. Cai no app shell autenticado.

### Requisitos de UI

- hero com proposta clara
- CTA acima da dobra
- seção curta explicando “feed your AI tools”
- linguagem simples e confiante

## Fluxo 2: primeiro acesso ao workspace

### Objetivo

Dar ao usuário uma sensação imediata de produto real, mesmo com pouca data.

### Passos

1. Usuário entra pela primeira vez.
2. Vê dashboard shell já elegante e funcional.
3. Recebe estado vazio orientado.
4. Entende os próximos passos: conectar integrações e começar a formar o brain.

### Requisitos de UI

- sidebar enxuta
- header limpo
- empty state premium
- checklist ou next actions simples

## Fluxo 3: conectar integração

### Objetivo

Permitir ao usuário iniciar a ingestão de contexto com pouco atrito.

### Passos

1. Usuário abre página de Integrations.
2. Vê cards de GitHub e Slack.
3. Entende status: connected, available ou coming soon.
4. Clica para conectar GitHub ou Slack.
5. Visualiza feedback de sucesso.
6. Retorna ao overview vendo que uma fonte foi adicionada.

### Requisitos de UI

- cards com hierarquia visual clara
- estados distintos
- CTA de conexão bem evidente
- feedback visual de sucesso

## Fluxo 4: visualizar o brain inicial

### Objetivo

Mostrar utilidade mesmo antes de um sistema avançado de IA estar pronto.

### Passos

1. Usuário entra no dashboard.
2. Vê overview com:
   - fontes conectadas
   - atividade recente
   - entidades ou projetos recentes
   - indicadores simples de cobertura
3. Entende que a Kogni está construindo memória operacional.

### Requisitos de UI

- cards simples e sofisticados
- activity feed enxuto
- linguagem de produto consistente
- evitar dashboards excessivamente densos

## Fluxo 5: entender valor do produto

### Objetivo

Fazer o usuário perceber rapidamente que Kogni não é só mais um dashboard.

### Passos

1. Usuário lê microcopy na landing e no app.
2. Percebe que o valor está em servir contexto às tools existentes.
3. Associa o produto a speed, memory e leverage.

### Requisitos de UI/copy

- usar o framing “We don’t compete with them. We feed them.”
- evitar jargão demais
- evitar parecer “search product” genérico
- enfatizar contexto vivo e aprendizado contínuo
