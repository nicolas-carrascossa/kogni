# UI Standards

## Princípio geral

A UI da Kogni deve transmitir sofisticação, clareza e confiança técnica.

Palavras-chave:

- premium
- clean
- moderna
- inteligente
- precisa
- calma

A interface não deve parecer um template SaaS genérico.

## Referências de sensação

- Linear
- Vercel
- Stripe

Não copiar literalmente. Usar apenas como referência de densidade, contraste, motion e polish.

## Regras visuais

### Layout

- muito espaço em branco
- grids simples
- hierarquia forte
- evitar excesso de blocos iguais
- preferir poucas áreas bem resolvidas a muitas seções fracas

### Tipografia

- headline forte e curta
- body text simples
- alta legibilidade
- evitar textos longos na interface
- usar contraste de peso mais do que excesso de tamanho

### Cor

- base neutra
- 1 cor primária de destaque
- acentos usados com moderação
- evitar visual muito colorido
- dark mode deve existir cedo

### Componentes

- bordas sutis
- radius consistente
- sombras leves
- estados vazios bonitos
- tabelas e listas simples
- cards apenas quando fizer sentido

### Motion

- microinterações suaves
- transições rápidas e discretas
- sem animações chamativas demais
- sensação de produto refinado

## Padrões por área

### Landing page

Deve ter:

- hero forte
- uma frase central memorável
- explicação curta do problema
- seção mostrando fontes e destinos
- CTA claro

Evitar:

- excesso de marketing fluff
- grids de features genéricas
- ilustrações sem função

### Login/Auth

Deve ter:

- visual limpo
- branding discreto
- foco em clareza
- poucos elementos

### Dashboard shell

Deve ter:

- sidebar minimalista
- header clean
- profundidade visual sutil
- empty states bem desenhados

### Integrations

Deve ter:

- cards compactos
- badges de status
- call to action direto
- “coming soon” bem resolvido

### Context overview

Deve ter:

- visão simples do estado do brain
- activity feed curto
- indicadores fáceis de entender
- aparência de infraestrutura inteligente

## Copy guidelines

Tom:

- confiante
- direto
- técnico sem exagero
- sem clichês de marketing

Boas direções de linguagem:

- company context
- memory layer
- live context
- connected sources
- operational memory
- feed your AI tools

Evitar:

- all-in-one platform
- revolutionize your workflow
- unlock productivity
- AI-powered everything

## Critério de qualidade

Antes de aceitar uma tela, verificar:

1. Parece produto premium ou template?
2. A proposta da Kogni está clara em poucos segundos?
3. Existe hierarquia visual real?
4. Há elementos demais?
5. A tela parece pronta para evoluir?
